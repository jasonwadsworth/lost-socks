1. **User uploads sock image** â†’ S3 (pre-signed URL)
2. **Frontend submits sock** â†’ Backend API â†’ EventBridge
3. **EventBridge triggers** â†’ Step Functions workflow
4. **Parallel agent analysis** â†’ Color, Size, Personality agents run simultaneously
5. **Sequential analysis** â†’ Historical agent queries past matches
6. **Final decision** â†’ Decision agent synthesizes all reports
7. **Match search** â†’ DynamoDB query for compatible socks
8. **Real-time updates** â†’ WebSocket broadcasts agent progress
9. **Results displayed** â†’ Full deliberation transcript with costs

## Why This Architecture?

Because matching `sock.color === otherSock.color` was simply too easy.

We needed:
- 5 AI agents with distinct personalities and expertise
- A formal consensus mechanism
- ISO compliance validation
- Psychological profiling
- Historical pattern analysis
- Monte Carlo simulations
- Real-time WebSocket updates
- 20+ EventBridge events
- Full audit trail
- Cost tracking per agent
